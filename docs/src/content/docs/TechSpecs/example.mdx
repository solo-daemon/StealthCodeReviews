## Usage of ZK-circuits

In the [example/verify_js](cci:7://file:///Users/theholygrail/startup/syntax-error/example/verify_js:0:0-0:0) folder contains a JavaScript implementation of a verification system for zero-knowledge proofs. This folder is part of a larger project that aims to provide a secure and private way to verify the correctness of computations without revealing the underlying data.

<Aside> In real world the issue creator will have to generate all this before starting an issue ,deploy the `verify.sol`</Aside>

### Folder Structure

### Key Components

### 1. `generate_witness.js`

This script generates a witness for a given input using the [witness_calculator.js](cci:4://file:///Users/theholygrail/startup/syntax-error/example/verify_js/generate_witness.js:0:0-21:0) module.

### 2. `input.json`

This file contains the input data for the verification process.

### 3. `proof.json`

This file contains the proof generated by the `generate_witness.js` script.

### 4. `public.json`

This file contains the public parameters for the verification process.

### 5. `verification_key.json`

This file contains the verification key used to verify the proof.

### 6. `verifier.sol`

This is a Solidity contract that verifies the proof on the Ethereum blockchain.

### 7. [witness_calculator.js](cci:4://file:///Users/theholygrail/startup/syntax-error/example/verify_js/generate_witness.js:0:0-21:0)

This module calculates the witness for a given input.

### Usage

To use the verification system, follow these steps:

1. Run `node generate_witness.js` to generate a witness for the input data in `input.json`.
2. The generated witness is stored in `proof.json`.
3. Use the `verifier.sol` contract to verify the proof on the Ethereum blockchain.
